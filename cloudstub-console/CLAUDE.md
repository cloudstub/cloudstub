# CloudStub Console — project instructions

## Workflow

- **Always create a branch before implementing an issue.** Never commit
  implementation work directly to `main`. Branch naming: `feature/<slug>` or
  `fix/<slug>` derived from the issue title (e.g. `feature/dashboard-service-filter`
  for issue #0003).
- Issues live in `issues/` and are numbered sequentially (`0001-`, `0002-`, …).
